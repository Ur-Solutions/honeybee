# Performance evidence

Raw benchmark reports in this directory are generated by
`scripts/bench-hsr-startup.mjs`. Compare reports only when their `config`
workload fields match for harness, account, concurrency, cwd, prompt, model,
timeout, poll interval, and stop event. The executable path and optional
`xBin` are expected to differ in a baseline/final comparison. Older reports do
not contain prompt/model; any comparison using them must disclose that audit
limitation. A stop-event mismatch also changes observation and cleanup timing,
even when both reports happen to contain the milestone being compared.

Use `--hive-bin <path>` for the full CLI and optionally `--x-bin <path>` for
the dedicated spawn entry. The full CLI remains the retirement command, so a
fast-entry report still cleans up each benchmark bee through `hive retire`.
Use `--model <id>` to pin a native model; otherwise reports record
`harness-default`.

The benchmark reports nearest-rank p50/p95. `request_to_turn_start_ms` is the
primary end-to-end Honeybee metric. `request_to_first_text_ms` includes model
provider latency and is deliberately excluded from Honeybee speedup claims.

See `HSR_STARTUP_RESULTS_20260720.md` for the controlled before/after summary.
